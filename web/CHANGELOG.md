### Fixed
- **Guide cover icons match content (#4, polish pass).** Sport guides already rendered the correct `CategoryGlyph`, but the cross-cut "Free to play" guide had no sport category — its `coverCategory` fell back to `scenic`, so it showed the scenic peak glyph. Added an optional `coverIcon` key to `GuideDto` (independent of category); `buildGuides()` tags the Free guide with `coverIcon: 'ticket'` (new `Ic.ticket`); `Guides.tsx` renders the `Ic` icon (white) when `coverIcon` is set, else falls back to `CategoryGlyph`. Sport guides keep their sport glyph; Viewpoint keeps the scenic glyph. No new tokens/colors; DTO additive (optional field), seam untouched.
- **Verified badge shows live relative time (#7, polish pass).** `PlaceDetailDto.verifiedAt` was free-text and `SpotDetail` printed it raw (`verified {p.verifiedAt}`) — the mock's "12 days ago" was static, and a real-API ISO string would have rendered as raw ISO. `verifiedAt` is now an **ISO 8601 timestamp** (DTO field semantics refined, shape unchanged); `SpotDetail` renders `formatDistanceToNow(new Date(p.verifiedAt), { addSuffix:true })` via new `date-fns` dep. Existing `{p.verifiedAt && …}` guard kept — absent timestamp = no badge. Mock RAW now carries real ISO dates on most spots (Löwenruh pitch + Pirita padel club left unset to demo the hidden-badge fallback). No new tokens/colors; seam untouched.
- **"Open in full map →" from spot detail (#10, polish pass).** Detail's underlying desktop map already flew + isolated the spot after #1, but there was no explicit affordance and no path from the (map-less) mobile detail. Added an "Open in full map →" link in `SpotDetail` (reuses `.fg-report-link`, `Ic.pin`) → `navigate('/explore?focus='+slug)`. `Explore` now reads `?focus=` → `focusSlug = detailSlug ?? params.get('focus')`, threaded as a new `focusSlug` prop into Desktop/MobileExplore → `SpotMap` (was passing `detailSlug` directly), reusing #1's `flyTo`/dim + cold-open handler. `focus` is one-shot — stripped from the nav search so it never sticks to spot links. Confirmed the map has no `interactive:false`/`scrollZoom.disable()` — pan/zoom already enabled. No new tokens/colors; DTO/seam untouched.
- **Search 0-results friendly empty state + "Submit this spot" (#11, polish pass).** `SearchBox` showed `Nothing matches "{q}"` with no CTA and no prefill path. Empty branch now renders **"No results for '{q}'"** plus a **"Submit this spot →"** button that `navigate('/add', { state:{ name:q }})`; `AddSpot.tsx` reads `useLocation().state.name` into the name input's initial value. New `.fg-search-cta` style (fg.css tokens only — display font, `--ink`, underline). No new tokens/colors; DTO/seam untouched.
- **"Near" triggers geolocation with loading + denied states (#8, polish pass).** Two inconsistent "near" controls: the `.fg-geobanner` called `request()` and showed "Locating…", but the `.fg-rail-head` "Near" sort button had **no `onClick`** — a dead click; and `geoStore.request()` fell back to Tallinn on denial **silently** (no user-facing signal). Now: `geoStore` gained a `denied` flag (set on the `getCurrentPosition` error callback / unsupported path alongside the curated fallback, cleared on locating/success) and fires a `toastStore` toast with the exact copy **"Location access denied — showing distances from Tallinn centre"**. The rail-head "Near" button (desktop + mobile) is wired to `request()`, disabled + labelled **"Getting location…"** while `status==='locating'`, and resets to "Near" otherwise. Threaded `onEnableLocation`/`locating` from `Explore` into `MobileExplore` (DesktopExplore already had them). No new tokens/colors; DTO/seam untouched.
- **Copy: bookmark wording + accurate rail-head noun (#9, #12, polish pass).** Saved empty state said "Tap the **stamp** on any specimen…" but the control is `SaveButton` — a bookmark glyph; copy now reads "Tap the bookmark icon on any spot to save it here." and the empty-state mark uses a new `Ic.bookmark` (matches the control) instead of `Ic.flag`. Explore rail-head said "{n} spots **nearby** · sorted by distance" even on the curated Tallinn-center default (no real GPS); noun is now conditional — "spots in view" when `curated` (geo not real), "spots nearby" only with real GPS. Threaded the existing `curated` boolean from `Explore` into `MobileExplore` (DesktopExplore already had it). Copy/icon only; no new tokens, DTO/seam untouched.
- **Guide/Saved card names wrap to 2 lines (#5, polish pass).** `.fg-card-body h3` is `white-space:nowrap` + ellipsis (1-line rail rhythm), so long names in the narrower `.fg-page-grid` cards (GuideDetail/Saved) were cut ("Kadrioru tennis co…"). Added a scoped `.fg-page-grid .fg-card-body h3` rule (`-webkit-line-clamp:2`, `white-space:normal`, `line-height:1.2`) — names now wrap to 2 lines, cards stay top-aligned (`align-items:start`). Rail single-line rhythm untouched. CSS-only; no new tokens.
- **Photo placeholder solid fill + big white glyph (#3, polish pass).** No-photo cards/hero rendered a category-tinted (`color-mix 32%`) background with a faint (`opacity:.55`) small category-colored glyph. Now `.fg-photo` fills solid `var(--pc)` and `.fg-photo-glyph` renders the `CategoryGlyph` in white at full opacity, centered. `Photo` gained a `large` flag (glyph 64 vs 26) so the detail hero shows a big icon while cards stay small; `color="#fff"`. Real-`url` branch untouched.
- **Search-jump pans the map (#2, polish pass).** Picking a spot from the top-bar search dropdown navigates to `/spot/:slug`, which feeds `focusSlug` (reusing #1's flyTo path) — no extra wiring needed for the in-Explore case. Closed one gap for "from any page": a search-jump from a non-Explore page (e.g. `/guides`) cold-mounts the map with `focusSlug` already set, so the `[focusSlug]` effect ran before the map existed and never re-fired → no fly. `SpotMap`'s `load` handler now applies the initial focus (`flyTo` to `focusRef.current`'s coords) once the style paints.
- **Spot-detail map flyTo + isolation (#1, polish pass).** Opening `/spot/:slug` left the shared Explore `SpotMap` viewport put — it only re-skinned the selected pin, never moved. `SpotMap` now takes a `focusSlug` prop (fed from `detailSlug` in `DesktopExplore`/`MobileExplore`, distinct from hover-driven `selectedSlug`): on change it `flyTo({zoom:15.5,duration:700})` to the item's coords and dims every other pin/cluster (`data-dim` → `opacity:.35` in `atoms.css`) while the focused pin stays enlarged via the existing `data-sel` styling. Closing the detail clears `focusSlug` → un-dims back to the multi-pin view. Also unblocks search-jump (#2) and "open in full map" (#10).
- **Map blank (P1, critical).** Spot pins/clusters never rendered on `/explore` or `/spot/:slug`. Two root causes, both in `widgets/map/SpotMap.tsx`, previously misattributed to the preview hidden-tab RAF throttle:
  1. **Stale `once('idle')` closure wiped the source.** The initial `items=[]` render registered `map.once('idle', apply)` capturing empty items; it fired *after* the populated `setData(10)` and reset the GeoJSON source to empty. Replaced with an `itemsRef` + a single `pushData()` that always reads the latest items, called from the `load` handler and the items effect — no stale closures.
  2. **GeoJSON source was never tiled.** MapLibre only generates tiles for a source referenced by ≥1 layer. Pins/clusters render as HTML `Marker` overlays driven by `querySourceFeatures`, with no GL layer on the source → tiles never built → `querySourceFeatures` always returned `[]` → zero markers. Added an inert `spots-probe` circle layer (radius/opacity 0) to force tiling.
- Verified live-style (mock seam): basemap renders (Tallinn, districts, fg monochrome) and 10 spots render as 5 pins + 1 cluster with correct taxonomy colors. Existing WebGL-detect / 10s-timeout / `fg-maperr` Retry fallback unchanged.
